# AtmosFlow Pricing — Group B Rollout

## Tiers (effective post-rollout)

| Tier      | Credits/mo | Monthly | Annual    | Stripe? | Notes                                       |
|-----------|-----------:|--------:|----------:|---------|---------------------------------------------|
| Free      | 1          | $0      | $0        | no      | Lower trial friction; watermarked output    |
| Solo      | 50         | $129    | $1,290    | yes     | Independent assessors                       |
| Pro       | 200        | $329    | $3,290    | yes     | Active consulting work — most popular       |
| Practice  | 500        | $749    | $7,490    | yes     | Small consulting practices                  |

Annual saves ≈ 17% (annual = 10× monthly; monthly × 12 = 12× monthly).

The previous **Team / Enterprise** tier is gone. Migration 009 rewrites
any existing `plan IN ('team','enterprise')` rows to `'practice'`. No
behavior change for those customers — same credits, same access.

## Credit accounting

- Credits are consumed on **finalize-assessment** and on **AI narrative
  generation**. Drafts, review, navigation are always free.
- **Free** tier: 1 credit, replenished by daily cron. The cron is
  idempotent: rows already at ≥1 are unchanged.
- **Paid monthly**: each Stripe `invoice.paid` event grants the tier's
  credits (handled in `api/webhook.js`).
- **Paid annual**: Stripe only fires `invoice.paid` once per year.
  Monthly credit grants come from `scripts/cron-monthly-credit-grant.ts`,
  scheduled on the 1st of each month at 00:00 UTC. Idempotency is
  enforced by `credits_ledger.reference_id = 'monthly-grant-YYYY-MM'` —
  a second run in the same month grants nothing.

## Free-tier UX

- Signup creates a `profiles` row with `plan='free'`,
  `credits_remaining=1`, `free_tier_signup_at=NOW()`,
  `billing_period='monthly'`. The bootstrap is in
  `src/utils/supabaseStorage.js` (`signUp`) with the canonical helper at
  `lib/free-tier.ts`.
- The user can finalize one assessment; the second attempt returns 402
  Payment Required from `/api/credits` and the SPA opens
  `src/components/pricing/PricingSheet.jsx`.
- Free-tier docx output carries a watermark — see "Watermark policy"
  below.

## Annual prepay flow

`/api/checkout` accepts `{ plan, billing_period, userId, userEmail,
returnUrl }` and creates a Stripe Checkout session in
`mode: 'subscription'`. The price is looked up from
`STRIPE_PRICE_<TIER>_<PERIOD>` env vars (set on Vercel after running
`scripts/setup-stripe-products.ts`).

On `checkout.session.completed`:
1. `profiles.plan` becomes the chosen tier.
2. `profiles.billing_period` becomes `'monthly'` or `'annual'`.
3. `profiles.subscription_status` becomes `'active'`.
4. `profiles.annual_renewal_at` becomes `NOW + 365 days` for annual
   subscriptions; null for monthly.
5. `TIER_CREDITS[plan]` are added to `credits_remaining`.
6. A `credits_ledger` row is written with `reason='subscription_grant'`.

## Customer Portal (self-serve)

`/api/customer-portal` returns a Stripe Billing Portal session URL for
authenticated paid users. Configure in Stripe Dashboard → Billing →
Customer Portal:
- Allow cancellation: yes (cancel-at-period-end)
- Allow plan changes: yes — between Solo/Pro/Practice, monthly+annual
- Allow payment-method updates: yes
- Allow invoice viewing: yes

Lifecycle handled by `api/webhook.js`:
- `customer.subscription.updated` with `cancel_at_period_end: true`:
  set `subscription_status='canceling'`, **keep plan and credits**
  through the paid period.
- `customer.subscription.deleted`: revert to free tier
  (`plan='free'`, `subscription_status='free'`, `credits_remaining=1`,
  `billing_period='monthly'`, `annual_renewal_at=null`).

The free-tier user (no `stripe_customer_id`) gets 404 from the portal
endpoint — there's no subscription to manage.

## Watermark policy

Free-tier docx output carries:
1. **Page header on every page**: `GENERATED WITH ATMOSFLOW FREE TIER`
   in light gray (RGB 200,200,200), centered, 9pt.
2. **Page footer on every page**: `Generated by AtmosFlow Free —
   atmosiq.prudenceehs.com — Upgrade for unrestricted reports`.
3. **Cover-page closing line**: `Free Tier Sample Report — for
   evaluation use only`.

Paid tiers render with **no watermark** and **no upsell language** — a
paid client deliverable must be indistinguishable from a non-AtmosFlow
report header/footer.

The interface contract is at `src/engine/report/watermark.ts`; the
docx-side implementation is in `src/components/docx/watermark.js`. The
`tier` flag is threaded into `generateConsultantDocx` via the `data`
argument as `data.watermarkConfig = { tier: profile.plan === 'free' ?
'free' : 'paid' }`. The caller is responsible for resolving the tier
from the user's profile.

## VIP override (raise a customer's daily limit)

Not implemented in this rollout (Group A's narrative rate limit is
fixed at 10/min, 100/24h, 5/24h-free). To temporarily raise a daily cap
for a specific customer, the cleanest path is to add a per-row override
column (e.g. `profiles.narrative_daily_override INT NULL`) and have
`api/narrative.js` honor it before the global `PER_DAY_LIMIT` check.
File a ticket against this rollout for that follow-up.

## Stripe products setup

```
STRIPE_SECRET_KEY=sk_test_… npx tsx scripts/setup-stripe-products.ts
```

Idempotent: the script lists existing products, matches by name, and
either UPDATES metadata in place or creates new products + recurring
prices. The output is a snippet of `STRIPE_PRICE_*` env-var assignments
ready to paste into Vercel project settings.

The script refuses to run against `sk_live_` without `--confirm-live`.

## Cron jobs

| Job                                | Schedule                  | Wired at                      | Idempotent |
|------------------------------------|---------------------------|-------------------------------|-----------|
| Free-tier 1-credit reset           | `0 0 * * *` daily 00:00 UTC | `/api/cron-free-tier-reset`     | yes       |
| Monthly credit grant (annual subs) | `0 0 1 * *` 1st-of-month  | `/api/cron-monthly-credit-grant` | yes       |

Both endpoints check `Authorization: Bearer ${CRON_SECRET}` if
`CRON_SECRET` is set (Vercel cron sets this header automatically when
configured).

## Migration path

- `supabase/migrations/009_pricing_rollout.sql` adds the new columns
  (`billing_period`, `annual_renewal_at`, `free_tier_signup_at`),
  rewrites `plan IN ('team','enterprise')` to `'practice'`, and adds
  CHECK constraints so the only valid plans are `free`/`solo`/`pro`/
  `practice`.
- Existing customers on legacy plans are auto-migrated by the
  `UPDATE` statement in 009.
- New signups land on `plan='free'` via `lib/free-tier.ts` + the
  `supabaseStorage.signUp` post-signup hook.
